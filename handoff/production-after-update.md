# XXXBER 本番更新後チェック

## 更新前に必ず残すもの

- 本番スプレッドシートを丸ごとコピーしたバックアップ
- 受け取ったCSVバックアップ
- 反映前のGitコミットまたはGitHub上の直前状態

CSVは売上の復元には使えますが、未確定伝票、スタッフ、メニュー、前借り、金庫、経費までは完全に戻せない場合があります。できればスプレッドシートのコピーも残してください。

## 更新直後にやること

1. GitHub Pagesの本番URLを開く。
2. 画面を強制更新する。
3. ログイン後、ヘッダーに同期エラーが出ていないことを確認する。
4. `↻ 再同期` を押して、同期完了になることを確認する。
5. テスト用の未確定伝票を1件作る。
6. 別端末または別タブでも未確定伝票を1件作る。
7. 片方を保存しても、もう片方の未確定伝票が消えないことを確認する。
8. 会計確定を1件だけ試し、保存後に本番スプレッドシート側へ反映されることを確認する。

## 現場へ伝えること

- 画面に `未同期 N件` が出たら、保存先への送信に失敗しています。
- `未同期 N件` を押すと再送します。
- 再送しても残る場合は `JSON出力` を押して、管理者に渡してください。
- `同期エラー（保存停止中）` が出ている端末では入力を続けず、`↻ 再同期` を押してください。

## 今回の修正範囲

- 保存APIのHTTPステータスとJSON本文を確認する。
- `success:true` 以外は保存成功扱いにしない。
- 保存失敗時は未同期データとして端末に退避する。
- 未確定伝票は保存前にサーバー最新版を取得し、ID単位でマージする。
- CSV復元データや初期化処理は本番には入れない。

## 残る構造リスク

Google Sheets/GASに配列丸ごと保存する構造は残っています。今回の修正で事故率は下がりますが、完全な同時編集対策にはSupabaseなどの行単位保存への移行が必要です。
